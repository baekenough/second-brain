# Guide: Elements of Style

Writing clarity guidelines from Strunk and White's "The Elements of Style" — applied to technical writing and AI agent communication.

## Overview

The Elements of Style guide adapts classic writing principles for technical documentation and agent communication patterns. It focuses on clarity, brevity, and active voice — the same principles that make code comments and architecture documents readable.

## Key Principles

- **Omit needless words**: Every word must earn its place
- **Use active voice**: "The agent reads the file" not "The file is read by the agent"
- **Put statements in positive form**: "Use concrete nouns" not "Avoid abstract language"
- **Write with nouns and verbs**: Strong subjects, strong predicates
- **Prefer specific to general**: Concrete over abstract, definite over vague
- **Place emphatic words at the end**: Most important information last in sentence

## Application

These principles apply directly to:
- Agent description fields (one-line summaries must be clear and specific)
- Wiki page bodies (concise synthesis, not verbose enumeration)
- Rule text (unambiguous, testable conditions)
