# R006: Agent Design Rules

**Priority**: MUST | Agent file format, required/optional frontmatter, model aliases, memory scopes, and separation of concerns.

## Overview

R006 defines the structure of agent files (`.claude/agents/{name}.md`). The critical design principle is **separation of concerns**: agent files contain role and capabilities only; skills contain implementation instructions; guides contain reference documentation. Violating this by embedding detailed instructions in agent bodies creates unmaintainable, duplicated content.

## Required Frontmatter

```yaml
name: agent-name           # Unique identifier (kebab-case)
description: Brief desc    # One-line summary for routing
model: sonnet              # sonnet | opus | haiku | opusplan
tools: [Read, Write, ...]  # Allowed tools
```

## Model Aliases

| Alias | Full ID | Use Case |
|-------|---------|----------|
| `haiku` | claude-haiku-4-5 | Fast, cheap tasks |
| `sonnet` | claude-sonnet-4-6 | General tasks (default) |
| `opus` | claude-opus-4-6 | Complex reasoning |
| `opusplan` | claude-opus-4-6 + plan mode | Architecture with approval gates |

## Memory Scopes

| Scope | Location | Git Tracked |
|-------|----------|-------------|
| `user` | `~/.claude/agent-memory/<name>/` | No |
| `project` | `.claude/agent-memory/<name>/` | Yes |
| `local` | `.claude/agent-memory-local/<name>/` | No |

## Separation of Concerns

| Location | Purpose |
|----------|---------|
| `.claude/agents/` | Role, capabilities overview, workflow |
| `.claude/skills/` | Detailed instructions, implementation rules |
| `guides/` | Reference documentation, best practices |
