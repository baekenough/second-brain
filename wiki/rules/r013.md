# R013: Ecomode Rules

**Priority**: SHOULD | Token efficiency mode for batch operations — compact output, input context pruning, and task-type-aware context budget management.

## Overview

R013 manages token budget through two complementary mechanisms: output compression (agents return summaries, not verbose results) and input context pruning (removing irrelevant retrieved chunks mid-task). Context budget thresholds are task-type-aware — research tasks trigger ecomode earlier (40%) than management tasks (70%).

## Activation

Auto-activates when: 4+ parallel tasks, batch operations, 80%+ context usage (default), or explicit "ecomode on". Override with "ecomode off", "verbose mode", or "show full details".

## Task-Type Thresholds

| Task Type | Context Trigger |
|-----------|----------------|
| Research (/research, multi-team) | 40% |
| Implementation | 50% |
| Review | 60% |
| Management | 70% |
| General (default) | 80% |

## Compact Output Format

```
[Batch Complete] {n}/{total}
├── {agent}: ✓/✗/⚠ {summary}
```

Compression rules: file lists → count only (unless < 5), error traces → first/last 3 lines, code → path:line ref only.

## Input Context Pruning

| Strategy | When | Behavior |
|----------|------|----------|
| Retain | Directly relevant | Keep as-is |
| Summarize | Background context | 1-2 line summary |
| Drop | Search noise, duplicates | Remove entirely |

Pruning is irreversible — generate summary BEFORE dropping original.
