# R007: Agent Identification Rules

**Priority**: MUST | Every response must begin with an agent identification header. Enforced via PostCompact hook and CLAUDE.md injection.

## Overview

R007 ensures users always know which agent is active. The identification header must appear at the start of every response — even after context compaction. The PostCompact hook re-injects this rule to prevent compaction amnesia.

## Full Format

```
┌─ Agent: {agent-name} ({agent-type})
├─ Skill: {skill-name} (if applicable)
└─ Task: {brief-task-description}
```

Default (no specific agent): `┌─ Agent: claude (default)`

## Simplified Format

For brief responses: `[mgr-creator] Creating agent structure...`

With skill: `[fe-vercel-agent → react-best-practices] Analyzing...`

## Routing Context

| Context | Identification |
|---------|---------------|
| No routing active | `claude (default)` |
| secretary-routing | `claude (secretary-routing)` |
| Skill invocation | `claude → {skill-name}` |

## Skill Invocation Format (Correct)

```
┌─ Agent: claude → {skill-name}
└─ Task: {brief-task-description}
```

Skill name MUST be integrated into the identification block, not displayed separately.

## Enforcement

- Advisory PostToolUse hook warns on missing identification
- PostCompact hook re-injects after context compaction
- Hard enforcement candidate if violation rate exceeds 20% (see [[r021]])

## Relationships

- **Tool ID**: [[r008]] for per-tool-call identification
- **Delegation**: [[r010]] — subagents include identification in their responses
- **Enforcement policy**: [[r021]] defines hook tiers
