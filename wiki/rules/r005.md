# R005: Optimization Guide

**Priority**: MAY | Efficiency strategies for token usage, task batching, and when (not) to optimize.

## Overview

R005 provides guidance on efficiency at three levels: parallel I/O execution, token minimization, and task batching. The core principle is "readability over optimization" — micro-optimize only when there's a measurable bottleneck, not preemptively.

A notable advanced technique: HTML comments in auto-injected `.md` files are hidden from the model during auto-injection but visible via the Read tool, enabling metadata embedding without token cost.

## Efficiency Strategies

| Strategy | When | Example |
|----------|------|---------|
| Parallel | 3+ independent I/O tasks | Read multiple files simultaneously |
| Caching | Same data accessed repeatedly | Cache file contents, reuse search results |
| Lazy Loading | Large datasets, partial use | Read only needed files |

## Token Optimization

- Include only necessary info, remove duplicates
- Summaries over full file contents when context is sufficient
- Concise expressions, minimize code block size

## Task Optimization

- **Batch**: Group similar tasks (edit 10 files at once)
- **Incremental**: Process only changed parts

## Context Optimization (v2.1.72+)

HTML comments in CLAUDE.md and rules/*.md files are hidden during auto-injection but visible via Read tool. Use for: metadata tags, validation checksums, conditional context, rule detail hiding.

## When to Optimize

| Do | Don't |
|----|-------|
| Repetitive tasks with measurable bottleneck | One-time tasks, already fast enough |
| Clear performance gain | When complexity exceeds benefit |
