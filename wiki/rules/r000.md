# R000: Language & Delegation Policy

All user-facing communication uses Korean; all file contents, code, and commits use English. The user delegates ALL file operations to the AI agent and does not edit files directly.

## Overview

This rule establishes two fundamental contracts: the output language split and the delegation model. Korean I/O ensures the user can communicate naturally while the codebase remains in standard English. The delegation model means the agent is the sole actor on the filesystem — the user provides intent, the agent executes.

## Key Details

- **Priority**: MUST
- **Enforcement**: Prompt-based (CLAUDE.md injection)
- Language by context:
  - User communication → Korean
  - Code, file contents, commits → English
  - Error messages to user → Korean
  - PR title/body, GitHub issues → Korean (default)
- Delegation flow: `User → (Korean prompt) → Agent → (file operations in English)`
- File naming conventions:
  - Rules: `{PRIORITY}-{name}.md` (e.g., `MUST-safety.md`)
  - Agents: kebab-case `.md` (e.g., `lang-golang-expert.md`)
  - Skills: `SKILL.md` inside kebab-case directory

## Integration

- **Interacts with**: [[r007]] (agent ID in Korean context), [[r008]] (tool calls in English), [[r010]] (all file writes delegated)
- **Triggered by**: every response and every file operation
- **PR language**: Korean by default; overridable in project CLAUDE.md
