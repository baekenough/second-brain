# R009: Parallel Execution Rules

**Priority**: MUST | 2+ independent tasks must execute in parallel using concurrent agent spawning, not sequential execution.

## Overview

R009 prevents the common anti-pattern of sequential execution for independent tasks. When tasks have no shared mutable state and no sequential dependencies, they MUST be parallelized. The rule includes an Agent Teams gate (R018) that must be evaluated before spawning 2+ parallel agents.

## Detection Criteria

**Independent (MUST parallelize)**:
- No shared mutable state between tasks
- No sequential dependencies
- Each completes independently

Examples: creating multiple agents, reviewing multiple files, batch operations on different resources.

## Agent Teams Gate

Before spawning 2+ parallel agents, evaluate R018 eligibility.

**Quick rule**: 3+ agents OR review cycle OR 2+ issues in same batch → use Agent Teams (when enabled)

## Concurrency Limits

| Rule | Value |
|------|-------|
| Max concurrent | 5 (soft default: 4) |
| Orchestrator | Must stay singleton (not parallelizable) |
| Large tasks (>3 min) | MUST split into parallel sub-tasks |
