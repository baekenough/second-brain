# R011: Memory Integration Rules

**Priority**: SHOULD | Native auto-memory primary; claude-mem MCP supplementary for cross-session search. Dual-system save protocol at session end.

## Overview

R011 establishes the memory architecture: native auto-memory (`memory:` frontmatter field) handles in-session and per-project persistence; claude-mem MCP handles cross-session search and temporal queries. The rule specifies a clear division: use native auto-memory first; only reach for claude-mem when native cannot satisfy the need.

## Memory Scopes

| Scope | Location | Git Tracked |
|-------|----------|-------------|
| `user` | `~/.claude/agent-memory/<name>/` | No |
| `project` | `.claude/agent-memory/<name>/` | Yes |
| `local` | `.claude/agent-memory-local/<name>/` | No |

First 200 lines of MEMORY.md loaded into system prompt automatically.

## Session-End Protocol

**Responsibility split:**

| Action | Owner |
|--------|-------|
| MEMORY.md update | sys-memory-keeper |
| claude-mem MCP save | Orchestrator (MCP tools are orchestrator-scoped) |
| episodic-memory | Auto-indexed after session — no action needed |

**Both steps must complete before confirming session-end to user.**

## When to Use claude-mem

| Scenario | Native | claude-mem |
|----------|--------|------------|
| Agent learns project patterns | Yes | — |
| Cross-session search | — | Yes |
| Temporal queries | — | Yes |
| Cross-agent sharing | — | Yes |
