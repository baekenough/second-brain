# R003: Interaction Rules

**Priority**: SHOULD | Response principles, status format, and output style selection (concise/balanced/explanatory).

## Overview

R003 defines how agents communicate: brevity over verbosity, clarity over abstraction, transparency about actions. The output style system maps task effort level to response depth, preventing both over-explanation (token waste) and under-explanation (lost context).

## Status Format

```
[Start] {task name}
[Progress] {current step} ({n}/{total})
[Done] {task name} — Result: {summary}
[Failed] {task name} — Cause: {reason} — Alternative: {solutions}
```

## Output Styles

| Style | Trigger | Behavior |
|-------|---------|----------|
| `concise` | effort: low, batch ops, ecomode | Key result only, no preamble |
| `balanced` | effort: medium, general tasks | Summary + key details |
| `explanatory` | effort: high, complex/learning tasks | Full reasoning, trade-offs |

### Style Priority

1. User explicit request → override
2. Ecomode active → force concise
3. Agent effort level → map to style
4. Default → balanced

## Request Handling

| Type | Action |
|------|--------|
| Clear | Execute immediately |
| Ambiguous | `[Confirm] Understood "{request}" as {interpretation}. Proceed?` |
| Risky | `[Warning] This action has {risk}. Continue?` |
