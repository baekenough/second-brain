# R017: Sync Verification Rules

**Priority**: MUST | Full 5+3 round verification required before committing AND pushing. `mgr-sauron:watch` must pass before any `git push`.

## Overview

R017 prevents drift between the agent system definition (CLAUDE.md, agent files, skill files) and its documented state. Every structural change triggers a mandatory verification cycle. The sauron:watch verification is non-negotiable — mgr-gitnerd will refuse a push if sauron was not run.

## Verification Phases

| Phase | Rounds | Focus |
|-------|--------|-------|
| Phase 1 | 5 | mgr-supplier:audit + mgr-updater:docs cycles |
| Phase 2 | 3 | Workflow alignment, reference integrity, philosophy |
| Phase 2.5 | 1 | Documentation accuracy, slash commands, routing completeness |
| Phase 3 | 1 | Wiki sync (missing/stale pages, broken cross-refs) |

## Quick Verification Commands

```bash
ls .claude/agents/*.md | wc -l           # agent count
find .claude/skills -name "SKILL.md" | wc -l  # skill count
find guides -mindepth 1 -maxdepth 1 -type d | wc -l  # guide count
find wiki -name "*.md" ! -name "index.md" ! -name "log.md" | wc -l  # wiki count
```

## When Required

Any change to: agents, agent frontmatter, skills, guides, routing patterns, rules, wiki pages.

## Self-Check Before Push

Five YES/NO questions before committing; three before pushing. Any NO → wait until verification completes.

## Relationships

- **Executes**: [[mgr-sauron]] runs the verification
- **Git gate**: [[mgr-gitnerd]] refuses push without sauron pass
- **Wiki**: [[wiki-curator]] and [[r022]] — Phase 3 wiki sync verification
- **Improvement**: [[r016]] — rule violations trigger update → re-verification cycle
