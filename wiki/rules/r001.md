# R001: Safety Rules

**Priority**: MUST | Defines prohibited actions, required checks before destructive operations, and the stop-and-report protocol for violations.

## Overview

R001 establishes hard boundaries around dangerous operations. The rule is enforced at the hook level for some categories (`rule-deletion-guard.sh`) but primarily through prompt-based behavioral guidance. When a safety boundary is crossed, the agent must immediately stop all operations and preserve state before reporting.

## Prohibited Actions

| Category | Prohibited |
|----------|-----------|
| Data | Expose API keys/secrets/passwords, collect PII without consent, log auth tokens |
| File System | Modify system files (/etc, /usr, /bin), delete outside project, modify .env/.git/config without approval |
| Commands | `rm -rf /` or broad deletes, shutdown/restart, sudo/su, network config changes |
| External | Access URLs without approval, send user data externally, download/execute unknown scripts |

## Required Before Destructive Operations

1. Verify target (correct path/resource)
2. Assess impact scope (what else is affected)
3. Check recoverability (can it be reversed?)
4. Get user approval

## On Violation

1. Stop all operations
2. Preserve current state
3. Report: what was detected, why it's risky, what action was taken
4. Wait for instructions
