# R010: Orchestrator Coordination Rules

**Priority**: MUST | The orchestrator NEVER modifies files directly — ALL file modifications MUST be delegated to appropriate subagents.

## Overview

R010 establishes the core orchestration architecture: the main conversation is the sole orchestrator, and it operates read-only. It uses routing skills to delegate file-writing tasks to specialist subagents. This separation ensures the orchestrator maintains coordination clarity while specialists maintain domain expertise.

**Exception**: Agent Teams members can spawn sub-agents as local orchestrators within their scope.

## Delegation Table

| Task Type | Required Agent |
|-----------|---------------|
| Create agent | mgr-creator |
| Update external | mgr-updater |
| Audit dependencies | mgr-supplier |
| Git operations | mgr-gitnerd |
| Memory operations | sys-memory-keeper |
| Python/FastAPI | lang-python-expert / be-fastapi-expert |
| Go code | lang-golang-expert |
| TypeScript/Next.js | lang-typescript-expert / fe-vercel-agent |
| Architecture docs | arch-documenter |
| Docker/Infra | infra-docker-expert |
| Server deployment | infra-docker-expert |
| Database schema | db-supabase-expert |
| Wiki writes | wiki-curator |
| Unmatched | mgr-creator → dynamic agent creation |

## Exception: Read-Only Operations

Subagent NOT required for: reading files for analysis (Read, Glob, Grep only), simple searches, direct questions. Any file creation/modification/deletion must be delegated — no "too small to delegate" exception.

## Session Continuity

After restart/compaction: re-read CLAUDE.md immediately. All delegation rules remain active. Never write code directly from orchestrator.

## Relationships

- **Parallel**: [[r009]] — parallel agents still delegate to specialists
- **Verification**: [[r017]] — sauron checks orchestrator never writes files
- **Dynamic creation**: [[mgr-creator]] as fallback when no specialist exists
- **Git**: [[mgr-gitnerd]] for all git operations, [[wiki-curator]] for all wiki writes
