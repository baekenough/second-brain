# R019: Ontology-RAG Assisted Routing

**Priority**: SHOULD | Routing skills should use ontology-RAG's `get_agent_for_task` tool to enrich agent selection with suggested skills.

## Overview

R019 adds an ontology enrichment layer on top of static routing. After static routing selects an agent, the orchestrator calls `get_agent_for_task(query)` to get `suggested_skills`, which are then prepended to the spawned agent's prompt as contextual hints. This enrichment is advisory and never blocks routing on failure.

## Integration Pattern

```
Static routing → agent selected
  ↓
get_agent_for_task(original_query)
  ↓
Extract suggested_skills
  ↓
Prepend to spawned agent prompt:
  "Ontology context suggests these skills may be relevant: {suggested_skills}"
```

## Failure Handling

MCP failure MUST NOT block or delay routing. Ontology-RAG is advisory only.

| Scenario | Action |
|----------|--------|
| MCP server unavailable | Skip silently, proceed |
| Empty response | Proceed unmodified |
| Parse error | Skip silently, log warning |

## Scope

Applies to: secretary-routing, dev-lead-routing, de-lead-routing, qa-lead-routing.

## Interaction with R015

Ontology-RAG adds post-fix confidence context (0.30-0.40) but does not change R015's display behavior — confidence thresholds still determine confirmation requirements.
